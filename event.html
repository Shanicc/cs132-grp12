Event Detection

STRUCTURE AND MECHANISM OF MODEL

  • Input. The model takes the time-series data as input.
  • Preprocessing. The input data may undergo preprocessing steps such as feature extraction
  • Peak Detection. This layer performs the main event detection task by identifying peaks or significant changes in the time-series data. The general peak detection algorithm used in the given code snippet is based on the find_peaks function from the scipy.signal library. This algorithm identifies peaks in a given signal by examining properties such as peak height, width, threshold, distance, and prominence. It searches for local maxima that meet the specified criteria and marks them as detected peaks. The algorithm leverages various techniques, such as smoothing, differentiation, and thresholding, to identify significant changes or peaks in the signal. By extracting these peaks, the algorithm provides insights into the occurrence of notable events or peaks in the data.
  • Output. The final output of the model is a set of timestamps corresponding to peak dates that may have corresponding events

MODEL PARAMETERS

  • Height. The height parameter was set to 0.2. This choice indicates that peaks needed to have a minimum height of 0.2 in the signal to be considered valid peaks. This value was likely selected based on the characteristics of the data and the desired sensitivity to detect prominent peaks while filtering out smaller fluctuations or noise.
  • Width. The width parameter was set to 0.2. This value represents the minimum width a peak must have to be considered valid. By choosing this value, the algorithm focuses on detecting peaks with a minimum duration or width of 0.2, potentially filtering out narrower or shorter fluctuations in the signal.
  • Threshold. The threshold parameter was set to 2. This threshold determines the vertical distance between a peak and its neighboring samples. Peaks needed to exceed this threshold to be detected. This value was likely chosen to ensure that detected peaks are significantly higher than their surrounding samples, reducing the chances of false positives caused by minor fluctuations.
  • Distance. The distance parameter was set to 1. This choice indicates that neighboring peaks needed to have a minimum horizontal distance of 1 sample between them to be considered separate peaks. This value helps prevent the algorithm from detecting multiple peaks in close proximity and instead captures distinct events by enforcing a minimum gap between peaks.
  • Prominence. The prominence parameter was set to 3. This value represents the vertical distance between a peak and its lowest contour line. Peaks needed to have a prominence greater than 3 to be considered valid peaks. This choice emphasizes the prominence or significance of detected peaks, ensuring that they stand out from their surroundings and have a noticeable impact on the signal.
